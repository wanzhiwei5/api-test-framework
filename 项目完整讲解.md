## 1. 这个项目是做什么的？<a name="1"></a>

### 一句话说清楚

> 搭建了一套**自动化测试框架**，让 API 测试从"手动点"变成"自动跑"。

### 生活中类比

```
你开了一家小餐馆（你是程序员）

你写了菜单（API 接口）：
  - GET  /health    → 检查餐厅还活着吗
  - POST /notes     → 点菜（创建笔记）
  - GET  /notes     → 看所有菜（获取列表）
  - GET  /notes/1   → 看某个菜（获取单条）
  - PUT  /notes/1   → 修改某个菜（更新笔记）
  - DELETE /notes/1 → 撤掉某个菜（删除笔记）

❌ 以前：每次改菜单，你要自己尝一遍所有菜（手动测试）

✅ 现在：雇了一个试菜机器人（自动化测试框架）
   每次改完菜单，机器人自动尝一遍
   哪个菜有问题，机器人马上喊你
```

| 测试速度 | 全部跑完只需约 2 秒 |
| 自动化程度 | 0 人工干预，git push 即触发 |

## 3. app/main.py 详解 — 被测试的 API <a name="3"></a>

### 这个文件是干嘛的？

它是**被测试的目标**。你的测试代码就是去检查这个 API 有没有正常工作。

就像打靶需要先有个靶子，这个文件就是你的靶子。

### 完整代码（标注了每一段的作用）

```python
"""
被测试系统 — 简单的笔记 REST API
"""
# ===== ① 导入工具 =====
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field
from typing import Optional
import uuid

# ===== ② 启动应用 =====
app = FastAPI(title="笔记 API", version="1.0.0")

# ===== ③ 数据模型（三个模具） =====

class NoteCreate(BaseModel):  # 创建笔记时用的模具
    title: str = Field(..., min_length=1, max_length=100)
    content: str = Field(..., min_length=1, max_length=1000)

class NoteUpdate(BaseModel):  # 更新笔记时用的模具（字段可选）
    title: Optional[str] = Field(None, min_length=1, max_length=100)
    content: Optional[str] = Field(None, min_length=1, max_length=1000)

class Note(BaseModel):  # 返回给用户时用的模具
    id: str
    title: str
    content: str

# ===== ④ 数据存储（本子） =====
notes_db: dict[str, Note] = {}

# ===== ⑤ 六个 API 接口 =====

# 接口1：健康检查
@app.get("/health")
def health_check():
    return {"status": "ok"}

# 接口2：创建笔记
@app.post("/notes", response_model=Note, status_code=201)
def create_note(note: NoteCreate):
    note_id = str(uuid.uuid4())[:8]
    new_note = Note(id=note_id, title=note.title, content=note.content)
    notes_db[note_id] = new_note
    return new_note

# 接口3：获取笔记列表
@app.get("/notes", response_model=list[Note])
def list_notes():
    return list(notes_db.values())

# 接口4：获取单条笔记
@app.get("/notes/{note_id}", response_model=Note)
def get_note(note_id: str):
    note = notes_db.get(note_id)
    if not note:
        raise HTTPException(status_code=404, detail="笔记不存在")
    return note

# 接口5：更新笔记
@app.put("/notes/{note_id}", response_model=Note)
def update_note(note_id: str, update: NoteUpdate):
    note = notes_db.get(note_id)
    if not note:
        raise HTTPException(status_code=404, detail="笔记不存在")
    updated = Note(
        id=note.id,
        title=update.title if update.title else note.title,
        content=update.content if update.content else note.content,
    )
    notes_db[note_id] = updated
    return updated

# 接口6：删除笔记
@app.delete("/notes/{note_id}", status_code=204)
def delete_note(note_id: str):
    note = notes_db.pop(note_id, None)
    if not note:
        raise HTTPException(status_code=404, detail="笔记不存在")
```

### 逐段讲解

#### ① 导入工具

```python
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field
from typing import Optional
import uuid
```

| 这行代码 | 大白话 |
|---------|--------|
| `FastAPI` | 帮你用 Python 写 API 的框架 |
| `HTTPException` | 返回错误信息，比如"404 找不到" |
| `status` | HTTP 状态码的编号，如 200=成功, 404=找不到 |
| `BaseModel` | 数据模型模板，定义数据格式 |
| `Field` | 给数据加规则，如"标题不能超过100字" |
| `uuid` | 生成唯一 ID，给每条笔记一个编号 |

#### ② 启动应用

```python
app = FastAPI(title="笔记 API", version="1.0.0")
```
翻译：开了一家叫"笔记 API"的餐厅，版本号 1.0。

#### ③ 三个数据模型

| 模具 | 什么时候用 | 标题 | 内容 | ID |
|------|-----------|------|------|----|
| NoteCreate | 创建笔记时 | ✅ 必须有 | ✅ 必须有 | ❌ 自动生成 |
| NoteUpdate | 更新笔记时 | ⭕ 可选 | ⭕ 可选 | ❌ 不用填 |
| Note | 返回给用户 | ✅ 有 | ✅ 有 | ✅ 有 |

```python
title: str = Field(..., min_length=1, max_length=100)
```
拆解：`标题` `类型是字符串` `必填` `最少1字` `最多100字`

```python
title: Optional[str] = Field(None, min_length=1, max_length=100)
```
拆解：`标题` `可以有也可以没有` `默认没有` `有的话1~100字`

#### ④ 数据存储

```python
notes_db: dict[str, Note] = {}
```
创建了一个空字典用来存笔记。存进去后长这样：
```python
{
    "a3f8c21d": Note(id="a3f8c21d", title="学习计划", content="1. 学Python"),
    "b4d9e32f": Note(id="b4d9e32f", title="购物清单", content="牛奶鸡蛋"),
}
```

注意：这是"内存数据库"，程序关闭数据就没了。

#### ⑤ 六个接口详解

**接口1：健康检查**
```python
@app.get("/health")
def health_check():
    return {"status": "ok"}
```
收到 GET /health → 返回 {"status": "ok"}
类比：打电话问"还营业吗？" → "营业！"

**接口2：创建笔记**
```python
@app.post("/notes", response_model=Note, status_code=201)
def create_note(note: NoteCreate):
    note_id = str(uuid.uuid4())[:8]      # 生成8位随机ID
    new_note = Note(id=note_id, ...)      # 组装成笔记
    notes_db[note_id] = new_note          # 存进本子
    return new_note                       # 返回结果
```
- 201 是"已创建"的专用状态码
- uuid4() 生成类似 "a3f8c21d-b4d9-..." 的字符串，[:8] 只取前8位

**接口3：获取笔记列表**
```python
@app.get("/notes", response_model=list[Note])
def list_notes():
    return list(notes_db.values())
```
把本子里所有的笔记拿出来，以列表形式返回。

**接口4：获取单条笔记**
```python
@app.get("/notes/{note_id}", response_model=Note)
def get_note(note_id: str):
    note = notes_db.get(note_id)
    if not note:  # 没找到
        raise HTTPException(status_code=404, detail="笔记不存在")
    return note
```
花括号 {} 表示变量。GET /notes/abc123 → note_id = "abc123"。
没找到返回 404。

**接口5：更新笔记**
```python
title=update.title if update.title else note.title
```
核心逻辑："有就改，没有就留原样"。
更新时你可以只改标题不改内容，反之亦然。

**接口6：删除笔记**
```python
note = notes_db.pop(note_id, None)
```
pop = 取出并删除，像从本子上撕掉一页。
204 = 删除成功，没有内容返回。

## 4. core/config.py 详解 — 配置管理 <a name="4"></a>

### 为什么需要这个文件？

> 搬家了要改地址，如果写在100个文件里要改100次。
> 集中写在一个文件里，改一次就行。

### 代码及讲解

```python
from pydantic_settings import BaseSettings

class TestConfig(BaseSettings):
    base_url: str = "http://localhost:8000"   # API地址
    timeout: int = 10                           # 超时时间(秒)
    report_dir: str = "reports"                 # 报告目录
    log_level: str = "INFO"                     # 日志级别

    class Config:
        env_file = ".env"                       # 也可从.env文件读
        env_prefix = "TEST_"                    # 环境变量前缀

config = TestConfig()  # 全局唯一配置对象
```

### 关键点

- `config = TestConfig()` 在文件末尾创建**全局唯一的配置对象**
- 其他文件用 `from core.config import config` 获取它
- 如果创建 `.env` 文件写 `TEST_BASE_URL=http://服务器.com`，会自动覆盖默认值

## 5. core/client.py 详解 — HTTP 客户端 <a name="5"></a>

### 为什么需要这个文件？

> 你不想每次吃饭都自己种菜、洗菜、做菜。
> 请一个厨师（client），你只要说"我想吃番茄炒蛋"。

### 代码及讲解

```python
import requests
from core.config import config
from core.logger import logger

class APIClient:
    def __init__(self, base_url=None):
        self.base_url = base_url or config.base_url
        self.session = requests.Session()
        self.session.timeout = config.timeout

    def _request(self, method, path, **kwargs):
        url = f"{self.base_url}{path}"       # 拼接完整地址
        logger.info(f"{method.upper()} {url}")  # 记录日志
        response = self.session.request(method, url, **kwargs)
        logger.info(f"← 状态码: {response.status_code}")
        return response

    def get(self, path, **kwargs):
        return self._request("GET", path, **kwargs)

    def post(self, path, json=None, **kwargs):
        return self._request("POST", path, json=json, **kwargs)

    def put(self, path, json=None, **kwargs):
        return self._request("PUT", path, json=json, **kwargs)

    def delete(self, path, **kwargs):
        return self._request("DELETE", path, **kwargs)
```

### 封装的好处

| 场景 | 没封装 | 封装后 |
|------|--------|--------|
| 发请求 | `requests.get("http://localhost:8000/health")` | `client.get("/health")` |
| 记日志 | 每行都写 print | 自动记录 |
| 改地址 | 每个测试都要改 | 只改 config 一处 |

### f-string 解释

```python
base_url = "http://localhost:8000"
path = "/health"
f"{base_url}{path}"  # 结果: "http://localhost:8000/health"
```

## 6. core/logger.py 详解 — 日志系统 <a name="6"></a>

### 代码及讲解

```python
import logging
import sys
from core.config import config

def setup_logger(name="api-test"):
    logger = logging.getLogger(name)
    logger.setLevel(config.log_level)
    handler = logging.StreamHandler(sys.stdout)
    formatter = logging.Formatter(
        fmt="%(asctime)s | %(levelname)-8s | %(message)s",
        datefmt="%H:%M:%S",
    )
    handler.setFormatter(formatter)
    if not logger.handlers:
        logger.addHandler(handler)
    return logger

logger = setup_logger()
```

## 7. tests/test_notes.py 详解 — 核心测试代码 <a name="7"></a>

这是**最重要的文件**，面试常问。

### 完整代码

```python
"""笔记 API 测试 — 覆盖 CRUD + 异常场景"""
import json
import pytest
import os
from core.logger import logger

# 加载测试数据
DATA_DIR = os.path.join(os.path.dirname(__file__), "..", "data")
with open(os.path.join(DATA_DIR, "test_data.json"), encoding="utf-8") as f:
    test_data = json.load(f)


class TestHealth:
    """健康检查接口测试"""
    def test_health_check(self, client):
        resp = client.get("/health")
        assert resp.status_code == 200
        assert resp.json() == {"status": "ok"}


class TestCreateNote:
    """创建笔记测试 — 含参数化（数据驱动）"""
    @pytest.mark.parametrize("note_data", test_data["valid_notes"])
    def test_create_note_success(self, client, note_data):
        resp = client.post("/notes", json=note_data)
        assert resp.status_code == 201
        body = resp.json()
        assert body["title"] == note_data["title"]
        assert body["content"] == note_data["content"]
        assert "id" in body
        pytest.last_note_id = body["id"]  # 存ID供后续测试

    @pytest.mark.parametrize("note_data", test_data["invalid_notes"])
    def test_create_note_invalid(self, client, note_data):
        resp = client.post("/notes", json={
            "title": note_data["title"],
            "content": note_data["content"],
        })
        assert resp.status_code == note_data["expected_status"]


class TestListNotes:
    """获取笔记列表测试"""
    def test_list_notes(self, client):
        resp = client.get("/notes")
        assert resp.status_code == 200
        assert isinstance(resp.json(), list)


class TestGetNote:
    """获取单条笔记测试"""
    def test_get_note_success(self, client):
        note_id = getattr(pytest, "last_note_id", None)
        if not note_id:
            pytest.skip("需要先创建一条笔记")
        resp = client.get(f"/notes/{note_id}")
        assert resp.status_code == 200
        assert resp.json()["id"] == note_id

    def test_get_note_not_found(self, client):
        resp = client.get("/notes/nonexistent")
        assert resp.status_code == 404
        assert "笔记不存在" in resp.json()["detail"]


class TestUpdateNote:
    """更新笔记测试"""
    def test_update_note_success(self, client):
        note_id = getattr(pytest, "last_note_id", None)
        if not note_id:
            pytest.skip("需要先创建一条笔记")
        update_data = {"title": "已更新", "content": "内容已修改"}
        resp = client.put(f"/notes/{note_id}", json=update_data)
        assert resp.status_code == 200
        body = resp.json()
        assert body["title"] == "已更新"
        assert body["content"] == "内容已修改"

    def test_update_note_not_found(self, client):
        resp = client.put("/notes/nonexistent", json={"title": "新标题"})
        assert resp.status_code == 404


class TestDeleteNote:
    """删除笔记测试"""
    def test_delete_note_success(self, client):
        note_id = getattr(pytest, "last_note_id", None)
        if not note_id:
            pytest.skip("需要先创建一条笔记")
        resp = client.delete(f"/notes/{note_id}")
        assert resp.status_code == 204
        # 验证确实删了：再查一次应该是404
        get_resp = client.get(f"/notes/{note_id}")
        assert get_resp.status_code == 404

    def test_delete_note_not_found(self, client):
        resp = client.delete("/notes/nonexistent")
        assert resp.status_code == 404
```

### assert 是什么？

```python
assert 1 + 1 == 2  # ✅ 对的，通过
assert 1 + 1 == 3  # ❌ 错的，测试失败
```
assert = "我断定……如果不对就报错"

### 参数化测试是什么？

```python
# 没用参数化：每多一条数据就要多写一个函数 ❌
def test_note_1(self):
    ...
def test_note_2(self):
    ...

# 用了参数化：自动用不同数据跑多次 ✅
@pytest.mark.parametrize("name", [数据1, 数据2])
def test(self, name):
    ...   # 自动跑2次
```

### 为什么存 pytest.last_note_id？

后面的测试（获取、更新、删除）需要知道"刚才创建的那条笔记的ID"。
这叫**测试间的数据传递**。

### 为什么用 getattr 而不是直接访问？

```python
note_id = getattr(pytest, "last_note_id", None)
# 安全地取属性，取不到就返回 None
```
因为创建测试可能没跑，直接 `pytest.last_note_id` 会报错。

### 为什么删除后还要再查一次？

```python
resp = client.delete(f"/notes/{note_id}")
assert resp.status_code == 204
# 拉一下门把手确认真的锁上了
get_resp = client.get(f"/notes/{note_id}")
assert get_resp.status_code == 404
```
测试思维的核心：**不仅要测操作本身，还要测操作的效果。**

```json
{

        {"title": "学习计划", "content": "1. 学Python\n2. 学pytest"},
        {"title": "购物清单", "content": "牛奶、面包、鸡蛋"}
    ],
    "invalid_notes": [
        {"title": "", "content": "标题为空", "expected_status": 422},
        {"title": "xxx...x(101个)", "content": "标题超长", "expected_status": 422}
    ]
}
```

| 分组 | 用途 | 期望结果 |
|------|------|---------|
| valid_notes | 正常数据（应该成功） | 返回 201 |
| invalid_notes | 非法数据（应该失败） | 返回 422 |

`\n` 表示换行。`expected_status` 是预期返回的状态码。

## 9. .github/workflows/test.yml 详解 — GitHub Actions <a name="9"></a>

### 代码及讲解

```yaml
name: API Tests

on:
  push:
    branches: [ "main" ]       # 推送代码到main时触发
  pull_request:
    branches: [ "main" ]       # 发起PR到main时触发

jobs:
  test:
    runs-on: ubuntu-latest     # 在Ubuntu虚拟机运行

    steps:
      - uses: actions/checkout@v4              # 1.下载代码
      - uses: actions/setup-python@v5          # 2.装Python
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt   # 3.装依赖
      - run: |
          uvicorn app.main:app --host 0.0.0.0 --port 8000 &  # 4.启动API(后台)
          sleep 3                                              # 5.等3秒
          pytest -v --html=reports/report.html                 # 6.跑测试
      - uses: actions/upload-artifact@v4       # 7.保存报告
        with:
          name: test-report
          path: reports/
          if-no-files-found: warn
```

### 流程图

```
你git push → GitHub收到代码 → 启动Ubuntu虚拟机
                               ↓
                     1. 下载你的代码
                     2. 装Python 3.11
                     3. pip install 装依赖
                     4. 启动笔记API（后台运行 &）
                     5. sleep 3 等启动完成
                     6. pytest 跑12条测试
                     7. 保存HTML报告
                               ↓
                     全部通过 → 绿色 ✅
                     有失败 → 红色 ❌
```

### & 符号

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 &
```
`&` = 后台运行。不加的话程序会卡在这里不继续执行。

### sleep 3

启动API需要1-2秒，不等它完全起来就跑测试会连不上报错。

## 10. conftest.py 详解 — 测试工具函数 <a name="10"></a>

```python
import pytest
from core.client import APIClient

@pytest.fixture(scope="session")
def client() -> APIClient:
    """整个测试会话共用一个客户端实例"""
    return APIClient()

@pytest.fixture(autouse=True)
def print_separator():
    """每个测试自动打印分隔线"""
    print()
    yield
```

| 部分 | 含义 |
|------|------|
| `@pytest.fixture(scope="session")` | 只创建一次，所有测试共用 |
| `def client()` | 测试函数可以接收 `client` 参数自动注入 |
| `@pytest.fixture(autouse=True)` | 每个测试自动执行，无需声明 |

## 12. 上传到 GitHub 的完整步骤 <a name="12"></a>

### Git 是什么？

> Git = 时光机
> - `git init` → 造一台时光机
> - `git add .` → 把文件放进时光机
> - `git commit` → 拍一张照片（版本快照）
> - `git push` → 把照片上传到云端（GitHub）

### GitHub 是什么？

> GitHub = 照片云盘

### 完整步骤

```bash
# 第一步：告诉Git你是谁（只做一次）
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

# 第二步：进入项目目录
cd api-test-framework

# 第三步：初始化Git（造时光机）
git init

# 第四步：把文件加入待上传
git add .

# 第五步：创建版本快照（拍照片）
git commit -m "初始化：接口自动化测试框架"

# 第六步：去GitHub创建仓库
# 打开 https://github.com/new
# 填仓库名 → 选Public → 不要勾README → 点Create repository

# 第七步：连接远程仓库
git remote add origin https://github.com/你的用户名/api-test-framework.git

# 第八步：改分支名
git branch -M main

# 第九步：推送到GitHub
git push -u origin main
# 会弹出登录窗口，选择"Sign in with your browser"
```

### 以后改代码后怎么上传

```bash
git add .
git commit -m "改了什么内容"
git push
```

### 如果 push 报错连不上

```bash
# 检查是否有代理
git config --global --list

# 如果用了代理软件，设置Git代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```
端口号看你的代理软件（Clash默认7890，V2Ray默认10808）。

## 13. 常见问题与解决 <a name="13"></a>

| 问题 | 原因 | 解决 |
|------|------|------|
| UnicodeDecodeError | Windows中文系统用GBK编码 | 打开文件时加 `encoding="utf-8"` |
| actions/upload-artifact@v3 报错 | 旧版本被弃用 | 升级到 `@v4`，加 `if-no-files-found` |
| git push 连不上 | 网络问题/代理 | 配置代理或检查网络 |
| ModuleNotFoundError | 没装依赖 | `pip install -r requirements.txt` |
| conftest.py 缩进错误 | 复制时多了空格 | 检查文件开头不能有空格 |
